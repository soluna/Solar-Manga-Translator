# 文本区域统一改造：影响评估

## 结论

手动添加、自动识别和合并生成的框应当是同一个领域实体：**文本区域（Text Region）**。它们只通过 `origin=automatic|user|derived` 描述来源，共用页面命令、翻译覆盖、排版、样式、移动缩放、撤销历史、嵌字和页面文档。

自动识别缓存与用户添加区域仍有不同的输入来源，这是合理的适配器差异，不应扩散成两套业务机制。当前项目状态 schema v2 中的 `manual_regions` 暂时作为旧项目和快照兼容适配器保留；页面文档中的统一区域集合才是审校工作台的公开事实。

## 影响矩阵

| 模块 | 影响 | 处理方式 |
|---|---|---|
| 自动识别 | 中 | 有效识别缓存继续权威；缓存缺失时从页面文档恢复，保留原区域 ID |
| 手动添加与单框 OCR | 高 | 新区域进入统一集合；初始字号继承邻近区域，OCR 无结果时不再回退到框短边 |
| 翻译与人工译文 | 低 | 所有来源继续使用同一 `update_translation` 命令和覆盖表 |
| 字体、字号和高级样式 | 中 | 所有来源使用相同解析函数；去除自动值额外 `-6px` 的预览分支 |
| 文本框移动与缩放 | 低 | 所有来源原本已共用命令；新增回归证明 bbox 更新一致 |
| 删除与撤销 | 高 | 当前用户添加/合并区域可物理删除；空集合明确表示删除，不再回退旧文档复活 |
| 自动框停用/恢复 | 低 | 维持可逆停用语义，避免把重新识别产生的区域误当永久用户删除 |
| 页面文档 | 高 | 统一集合负责按 ID 调和物化缓存、持久化自动区域和当前用户区域 |
| 可编辑缓存 | 中 | 变成可重建的派生物；结构编辑前确保缓存存在，避免后续嵌字缺少自动区域 |
| 空白页/擦除 | 低 | 不改变空白页像素和 artifact 依赖；区域布局修改仍使最终产物 stale |
| 重嵌字与导出 | 中 | 保持现有渲染入口；缓存恢复后的稳定 ID 保证译文和样式覆盖继续命中 |
| 项目恢复 | 中 | schema v2 无需迁移；旧 `manual_regions` 仍可恢复进统一页面集合 |
| 快照恢复 | 中 | 继续读取旧适配字段；删除兼容字段需等 page document 纳入版本化快照 |
| Classic/Canvas 审校 | 低 | 返回结构兼容，新增 `origin`；保留 `manual` 布尔值供旧前端兼容 |
| 调试接口 | 低 | `kind` 保留，新增 `origin`；原始 OCR 缓存仍可独立查看 |
| 性能 | 低 | 结构编辑在缓存缺失时多一次页面文档到缓存的恢复；正常有缓存路径不增加扫描 |

## 已实施的统一规则

1. 页面区域按 ID 组成一个有序集合；有效缓存存在时缓存权威。
2. 缓存缺失时保留持久化自动区域，并用“当前用户区域集合”整体替换旧用户区域。
3. 空用户区域集合具有明确含义：用户区域已全部删除。
4. 从页面文档恢复缓存时保留 `translation_region_key` 和 `style_region_key`。
5. 自动与用户区域通过同一页面命令编辑译文、字号、bbox 和样式。
6. 新增区域的建议字号优先继承同方向的邻近区域；没有参考区域时使用有上限的框尺寸比例。
7. 画布中编号、缩放控制点和样式按钮保持屏幕固定尺寸，不随页面缩放成为“大圆点/放大镜”。

## 兼容边界与后续迁移

本轮不直接删除 `manual_regions`。它仍被 schema-v2 项目状态、快照清单和恢复流程引用；无版本迁移地删除会让旧项目丢失用户添加区域。后续若升级到 project-state schema v3，应让快照显式包含页面文档修订或页面区域集合，迁移成功后再删除该适配字段。

本轮也不把“停用自动识别区域”改成物理删除。自动区域属于识别产物，重新识别会重新生成；用户真正需要的是从当前翻译/嵌字中排除并可撤销，现有停用/恢复语义更安全。

## 回归契约

- 缺少 `regions.json` 时：3 个自动区域 → 新增后 4 个 → 删除后恢复 3 个。
- 自动区域 ID 在缓存恢复后保持稳定，原译文和样式覆盖继续命中。
- 自动区域和用户区域执行相同译文、字号、bbox 命令后得到相同字段语义。
- 新增 45×45 用户区域在邻近区域字号为 18 时采用 18，而不是 45。
- 340% 定位后，缩放控制点不超过 14px，样式按钮不超过 30px。
